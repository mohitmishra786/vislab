# VisLab Widgets — Positioning (internal)

## One-liner

Embed live CPU, cache, OS, and storage simulations in technical docs — not screenshots or YouTube.

## Primary audience

- Technical bloggers (systems / infra / databases)
- CS educators and course site authors

## Secondary

- Company eng blogs (slower adoption, a11y/legal review)
- Textbook authors (export GIF/MP4 helps)

## Anti-audience

- General frontend designers (Excalidraw)
- Data scientists (Observable / D3)
- People who only need static flowcharts (Mermaid)

## Category

**Systems simulation embeds for MDX/HTML** — not “diagram library.”

## Aesthetic reference

PlanetScale-style interactive explainers — flat, monospace — without cloning article titles.

## Primary CTA (pre-npm)

1. Star the repo  
2. Clone + `pnpm run build` + Studio / demo-blog  
3. After publish: `pnpm add @vislab/react`
