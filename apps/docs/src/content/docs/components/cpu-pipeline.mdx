import { CpuPipeline } from "@vislab/react";

Instruction bubbles flow through configurable stages (default IF → ID → EX → MEM → WB). A simplified data-hazard detector can insert stall bubbles when EX and MEM share a label.

## Live demo

<figure>

  <figcaption>

  </figcaption>
</figure>

## When to use

Explaining pipelining, stalls, or multi-stage instruction flow in systems or architecture posts.

## Props

| Prop | Type | Default | Description |
| ---- | ---- | ------- | ----------- |
| `stages` | `string[]` | `IF,ID,EX,MEM,WB` | Stage labels |
| `autoPlay` | `boolean` | `true` | Start animating (respects reduced motion) |
| `themeName` | `string` | `dark-premium` | Theme id |

## Pedagogy notes

This is **not** a cycle-accurate CPU model. Hazard detection is intentionally simplified for teaching intuition. Cite a textbook or course notes alongside for rigorous definitions.

## Accessibility

Surround with a figcaption. Canvas has `role="img"` and a hidden summary; controls are keyboard-accessible.
